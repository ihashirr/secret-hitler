# Mobile Strategy

## Interaction model
- Keep the board readable on narrow portrait screens
- Use a top banner for status and a bottom desk for active decisions
- Avoid full-screen modal flows once the game has started

## Onboarding
- Join flow is split into alias selection, avatar selection, and room code entry
- Room creation should work without extra setup beyond opening the app

## UI direction
- Classified dossier aesthetic with paper tones, stamped accents, and vintage avatars
- Large tap targets and simple action verbs for high-friction moments like voting and executions

## Constraints
- Layouts target `100dvh`
- Most play happens on phones, so side-by-side desktop-only patterns should be avoided
